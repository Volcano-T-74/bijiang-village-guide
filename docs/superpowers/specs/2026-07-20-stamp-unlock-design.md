# 集章寻迹点亮交互设计

## 目标

将室内九点模拟到达与“集章寻迹”页面打通。游客第一次确认到达某个景点时，立即点亮对应邮章，并在“我的路线”页面看到可关闭的祝贺弹层。

## 状态模型

- 继续使用现有 `visitedSlugs` 作为邮章点亮的唯一状态来源。
- 不新增独立的 `unlockedStampSlugs`，避免足迹和邮章状态不同步。
- `visitedSlugs` 继续写入现有 Demo `localStorage` 状态，因此刷新后邮章仍保持点亮。
- 新增仅存在于当前页面内存的待展示邮章状态。刷新页面后不重复显示祝贺弹层。

## 到达流程

1. 用户在“我的路线”地图点击景点标记。
2. 用户点击“模拟到达此处”确认。
3. 系统在写入 `visitedSlugs` 前判断该景点是否已经访问。
4. 第一次访问时，更新当前位置、已访问状态和本地持久化状态，并显示该景点的邮章祝贺弹层。
5. 重复访问时，不再次显示祝贺弹层，保留现有普通到达提示。
6. 现有匿名 `simulated_arrival` 事件记录和偏离路线重规划逻辑保持不变。

## 集章寻迹页面

- 九枚邮章初始均显示锁定状态。
- 点亮状态完全根据 `visitedSlugs.has(place.slug)` 计算，不使用数组序号或硬编码数量。
- 顶部进度显示实际已点亮数量，例如 `1 / 9`。
- 进度圆点与九个景点一一对应；已访问点显示点亮样式。
- 锁定邮章使用灰度景点图和锁图标，辅助文本为“待解锁”。
- 已点亮邮章恢复彩色景点图，并显示“已点亮”印章。
- 点击锁定邮章提示“到达该景点后即可解锁”；点击已点亮邮章提示“这个印章已经点亮”。

## 祝贺弹层

- 弹层只在“我的路线”页面首次点亮邮章后出现。
- 内容包含对应景点名称、景点图和“恭喜你已点亮”文案。
- 提供明确的关闭图标按钮；点击遮罩也可关闭。
- 使用语义化 `role="dialog"`、可读标题和关闭按钮标签。
- 手机端采用底部浮层布局，桌面端居中显示；内容不得遮挡或溢出视口。
- 关闭只清除当前弹层状态，不影响邮章、足迹、路线或当前位置。

## 错误与边界

- 后台事件记录失败不影响本地点亮和弹层显示。
- 路线重新规划失败不撤销已点亮邮章。
- 重复到达已点亮景点不重复祝贺。
- 旧版已保存的 `visitedSlugs` 在升级后直接映射为已点亮邮章。
- 景点数据尚未加载时，集章页按空列表安全渲染。

## 测试

- 初始状态九枚邮章全部锁定，进度为 `0 / 9`。
- 第一次模拟到达后显示对应祝贺弹层，并能通过关闭按钮关闭。
- 进入“集章寻迹”后对应邮章显示已点亮，进度更新为 `1 / 9`。
- 刷新后邮章仍点亮，但祝贺弹层不自动重现。
- 重复模拟到达同一景点不重复显示祝贺弹层。
- 事件记录失败时邮章仍正常点亮。
- 现有定位、路线重规划和状态恢复测试继续通过。

## 非目标

- 不增加真实 GPS 自动打卡。
- 不增加独立二维码、NFC、蓝牙 Beacon 或 UWB 打卡。
- 不新增邮章后台模型或跨设备账号同步。
- 不修改现有九点路网和推荐路线算法。
