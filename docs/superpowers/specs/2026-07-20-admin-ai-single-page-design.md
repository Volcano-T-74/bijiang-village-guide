# 后台 AI 单页整合设计

## 目标

后台只保留“数据概览”这一主要工作页面。AI 运营分析不再拥有独立菜单或独立全屏页面，而是作为统计卡片下方、数据表格上方的一块紧凑对话区域。

## 页面结构

数据概览页面依次显示：统计卡片、AI 运营分析对话区、原有游客会话/足迹/收藏等数据表。AI 区域包含会话下拉选择、新建会话图标按钮、7/30/90 天统计周期、固定高度的消息列表、问题输入和发送按钮。

历史会话和数据库持久化能力继续保留，但不再使用占据整页的会话侧栏。原 `/admin/ai-analytics/` 地址重定向到 `/admin/data-overview/`，旧链接仍可使用。

## 错误诊断

DeepSeek 上游 HTTP 状态只转换为安全错误类别，不保存或显示响应正文：

- 401/403：密钥无效或无权限。
- 402：账户余额不足。
- 429：请求过于频繁。
- 5xx：DeepSeek 上游服务异常。
- 无 HTTP 状态：网络连接失败。
- 本地配置缺失、超时和响应格式异常继续使用既有分类。

后台对话区显示对应中文提示和重试按钮。密钥、上游正文和异常堆栈不进入页面或数据库。

## 验收

- SimpleUI 左侧菜单只有“数据概览”，没有独立“AI 运营分析”。
- `/admin/data-overview/` 同时显示原有统计/表格和紧凑 AI 对话区。
- 独立 AI URL 重定向到数据概览。
- 会话切换、连续追问、刷新恢复和失败重试仍可使用。
- 上游错误能区分密钥、余额、限流、网络和服务异常。
- 员工仍只能读取自己的对话记录。
