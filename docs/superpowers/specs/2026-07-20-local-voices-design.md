# 当地声音栏目设计

## 目标

在“碧江故事”页面增加独立的“当地声音”栏目，从后台读取当地人的讲述音频并提供在线播放。音频不与具体景点或景点故事绑定。

## 数据设计

新增 `LocalVoice` 模型，字段包括：

- 标题
- 原始文件名
- 音频文件地址
- 音频时长（秒）
- 语言代码与前端展示标签
- 后台排序
- 启用状态

音频文件作为项目静态资源存放于 `frontend/public/audio/`，数据库保存可访问的 `/static/audio/...` URL。这样本地 Vite、Django 静态文件和 Render 部署使用同一地址规则。后台支持查看、排序、启用和停用记录。

## 初始数据

将 `audio/` 中 5 个 `.m4a` 文件复制到前端静态音频目录并建立数据库记录：

- `国语版 顺德碧江村介绍.m4a`：普通话，88 秒
- `顺德碧江村介绍粤语版.m4a`：粤语，73 秒
- `中原路 4.m4a`：当地讲述，118 秒
- `20260719_223040.m4a`：乡音记录，202 秒
- `20260719_223446.m4a`：乡音记录，198 秒

导入命令必须可重复执行，按原始文件名更新或创建记录，不重复插入。

## API

新增只读接口 `GET /api/v1/local-voices/`，仅返回启用记录，并按后台排序、ID 排序。每项返回 ID、标题、文件 URL、时长、语言代码和语言标签。

## 前端交互

“碧江故事”页面在故事分类之后增加“当地声音”栏目。列表显示标题、语言标签和时长；点击播放按钮后原地播放，并显示播放或暂停状态、播放进度、当前时间和总时长。

同一时间只允许播放一条音频。切换条目时先暂停并释放上一条播放实例，再为新一代播放创建唯一的活动 `Audio` 实例，避免旧媒体事件污染当前状态。音频加载或播放失败时保留列表并显示简短错误提示。活动条目显示当前播放时间和总时长；页面销毁或切换视图时停止播放，避免后台继续发声。

## 视觉与无障碍

复用现有纸张色、棕色文字、蓝色信息色和圆形播放按钮。栏目使用一个完整列表而不是嵌套卡片。播放按钮提供可访问名称，进度控件提供音频标题，状态变化通过现有提示区域反馈。移动端保证按钮触控区域至少 44 像素，文字和时长不重叠。

- 在桌面与移动视口检查播放器布局和真实 `.m4a` 文件请求。
